### Summary
Implemented Tango Pi inbox/wake-up UX cleanup within the requested scope.

### Files changed
- `packages/tango/extensions/pi/index.ts`
  - Removed `tango_inbox(action: "handled"...)` from normal tango-message next actions.
  - Result items now point to `tango_result(...)`.
  - Blocked/ask items now point to `tango_activity(...) then tango_message(...)`.
  - Error/offline/stalled and other non-result items now point to `tango_activity(...)`.
  - Inbox poller now re-fetches unresolved inbox state immediately before sending a follow-up/card, preventing handled/dismissed/stale items from being delivered after reload/race.
- `packages/tango/src/inbox.ts`
  - Added `markActivityItemsInspected(meta)` consume-on-inspect behavior for non-result inbox items.
  - Current actionable items are marked `read`.
  - Stale/non-actionable items are marked `handled`.
  - Current `error` items remain visible/read while source status is still `error`.
  - Reused unresolved-state helper for result handling.
- `packages/tango/src/server.ts`
  - `GET /api/v1/runs/activity` now marks matching non-result inbox items read/handled by default.
  - Supports `peek=true` to inspect without consuming, matching the existing result peek pattern.
- `packages/tango/src/cli.ts`
  - Default `tango inbox` remains unresolved-focused.
  - Added explicit `--all` debug listing flag, sent as `include=all` to the API.
- `packages/tango/src/inbox.test.ts`
  - Added targeted tests for activity inspection consuming current blocked/error items as read and stale blocked attention as handled.
