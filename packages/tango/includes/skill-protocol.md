## Skill Protocol

If your role has available skills and one matches your assigned task, read that skill's `SKILL.md` before acting. Treat role-assigned skills as procedural guidance for relevant work, not optional background.

If the assignment explicitly names a skill, load it first. If no assigned skill matches the task, proceed normally.

Use skills within your assigned scope. A skill does not authorize expanding scope, delegating, mutating durable state, or using external systems unless your role, task, and project instructions allow it.

When a skill uses Loom in a multi-Loom project, confirm the target Loom from `LOOM_DEFAULT`, explicit `-L`, or `loom current` / `loom list` before writing durable updates.
