### Summary
Implemented the first shippable Tango A2A coordination cleanup slice in `packages/tango`.

### Changes
- `src/cli.ts`
  - Added `tango result --unread`, `--inbox <id>`, and `--peek`.
  - Result retrieval supports full, untruncated content in normal/text and JSON paths.
  - `collect-results` now fails fast as removed and points users to `tango result --unread`; help text no longer advertises it.
  - Added `tango wait` for explicit targets and repeated `--run-id`/`--run-dir`, with `--until` and `--mode any|all`.
- `src/server.ts`
  - Added `/api/v1/runs/wait` control-plane endpoint.
  - Added result `peek=true` support so reads can avoid marking result inbox items handled.
- `src/controlPlane.ts`
  - Added `waitRuns()` and wait condition evaluation for `terminal`, `result-ready`, `attention`, `blocked`, `error`, `settled`, and `inbox`.
- `src/lifecycle.ts`
  - Fixed terminal semantics: `blocked` is no longer terminal.
- `src/board.ts`
  - Added direct-child tree projection under `board.tree.directChildren`.
  - Added descendant aggregate counts, mode markers (`↔`/`→`), and delegation marker/capability (`L` for `lead`).
- `extensions/pi/index.ts`
  - Removed user-facing `tango_collect_results` Pi tool.
  - Enhanced `tango_result` to support target, unread, inbox, and peek retrieval without truncating result content.
  - Added `tango_wait` Pi tool.
  - Added a first compact live `tango-live` widget above editor, scoped to local/direct children and showing mode/L markers plus descendant aggregates only when nonzero.
- `src/a2a-cleanup.test.ts`
  - Added targeted coverage for blocked/non-terminal semantics, wait any/all behavior, and board direct-child aggregate projection.

### Validation
- Passed: `npm run check --workspace @bravo/tango`
- Passed targeted test after build: `npm run build --workspace @bravo/tango && node --test packages/tango/dist/a2a-cleanup.test.js`
- Re-ran check and targeted test successfully after final source edits.

### Notes / Scope Boundaries
- `tango wait` first slice covers explicit target names and repeated `--run-id` / `--run-dir`. Broader target-set selectors like descendants/workstream/root-session waits are not implemented yet.
- Pi live widget is intentionally minimal and uses the new board projection; richer overlay/detail behavior is left for a later slice.
- Existing unrelated working-tree items were present and not touched: role deletions, `.loom/`, and unrelated docs/feedback files.
