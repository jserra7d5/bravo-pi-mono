Compare alternatives in Loom and create or propose an explicit decision. Use Loom only if a Loom/node/inbox reference or Loom context is provided.

User input:

```text
$ARGUMENTS
```

Workflow:
1. Fetch context for the decision area and relevant variants.
2. Compare options against stated criteria, constraints, risks, and architecture smells.
3. Ask for missing decision criteria if needed.
4. Recommend a choice, defer, or request more research.
5. Record the decision with Loom commands when approved/appropriate, preserving rejected or deferred alternatives with rationale.

Do not erase alternatives just because one is chosen.
