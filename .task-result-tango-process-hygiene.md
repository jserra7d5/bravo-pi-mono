### Behavior implemented
- Server autostart now uses an atomic startup lock at `server.json.start.lock`.
  - The lock directory is created before acquisition.
  - The process that acquires the lock is the only process that spawns the server.
  - Waiters do not spawn competing servers; they poll discovery/readiness.
  - The startup lock is held until readiness/timeout and stale locks are reaped conservatively.
- Server compatibility/readiness probing now uses the cheap `/api/v1/health` endpoint and requires advertised `runs` capability instead of probing `/runs` and `/board`.
- `/api/v1/health` now advertises capabilities (`runs`, `board`, `inbox`).
- Server instances periodically self-retire by closing themselves if the discovery file no longer points to their own `{pid,url,token}`.
- Pi passive UI polling paths now have in-flight guards, subprocess timeouts, and exponential-ish backoff for:
  - footer status refresh
  - inbox poller
  - parent reconciler
  - live widget refresh
